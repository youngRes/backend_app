Chapter Up

data type to store the information related to a chapter of a video game.
chapterCode (optional)
String code that identifies a chapter inside a video game.
example: chapter-5
chapterDescription (optional)
String short description of a chapter.
example: Chapter where Paul ...
highlights (optional)
array[String] Tips for the teachers to understand what the results obtained can means
example: ["this chapter study empathy ....","low values on ..."]
snapshot (optional)
String url where the picture of the snapshot of the cahpter is stored
example: www.serverlocations.com/img/snp_3.png

Decision Up

data type to store the decision taken during an event
eventCode (optional)
String code that uniquely identifies an event
example: event-1-2-3
eventDescription (optional)
String description of the events where this decession was taken
example: in this event the student is asked for its favorite pet...
eventType (optional)
String type of the event (multiple choice, timed ...etc)
example: choice
choice (optional)
String String with the choice a student take in that event, depending on eventType this value can be numeric or a string. However the API always returns an string an latter conversion need to be made on the front-end.
example: cat
student (optional)
Student

Entry Up

Object used field of a Filter. Is used to indicate that a 'key' value of numeric (to know the type of Filter please refer to the Filter entry) must be between a 'min_value' and 'max_value'. In the case of being of type textual the max_value is ignored and the filter is equivalen of 'key == min_value'
key (optional)
String
example: age
min_value (optional)
String
example: 10
max_value (optional)
String
example: 13

EntryList Up

Object used as a entry of a general dictionary
key (optional)
String
example: answer1_choices
values (optional)
array[String]
example: ["cat","dog","fish","bird"]

Event Up

data type used to store an specific event inside a game
eventCode (optional)
String code that uniquely identifies an event
example: event-1-2-3
eventDescription (optional)
String short description of the event.
example: event that occurs after ...
highlights (optional)
array[String] Tips for the teachers to understand what the results obtained can means
example: ["Select fish means ....","low values on ..."]
eventType (optional)
String type of the event (multiple choice, timed ...etc)
example: choice
parentDecision (optional)
Decision
possibleChoices (optional)
array[String] List with all the possible chioces for the event if the type os 'multiple-choice' or with the min (1ยบ element) and max(2ยบ element) time if type is 'timed'.
example: ["cat","dog","bird","fish"]

Filter Up

data type used to retrieve the information related to a filter. Type can be numeric or textual, if textual values contains a list with all the possible values of that filter. If numeric, then the list contains two elements, the first one is the minimun and the second one is the maximum.
id (optional)
String name of the attribute that can be filtered
example: city
type (optional)
String numeric or textual
example: textual
values (optional)
array[String] contains all the possible values of that field if type is textual. If types is numeric contains two elements, the first one is the minimum, the second one is the maximum.
example: ["Madrid","Oulu"]

FilterApply Up

Contains a list of filter to be applied when retrieving decissions. Is a Json object with a &quot;group&quot;,&quot;student&quot; and &quot;test&quot; attributes that contains a List of &quot;Entry&quot; objects with the filters to applied to their respective fields. If any attribute (group, student or test) contains an empty list no filter is applied.
group (optional)
array[Entry]
student (optional)
array[Entry]
test (optional)
array[Entry]
